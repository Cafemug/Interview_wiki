```
Q. key의 종류?

A.
1. Candidate Key (후보키)
Tuple을 유일하게 식별할 수 있는 속성들의 부분 집합입니다

아래의 조건들을 모두 만족해야합니다
유일성 : Key로 하나의 Tuple을 유일하게 식별할 수 있음
최소성 : 꼭 필요한 속성으로만 구성

2. Primary Key (기본키)
후보키 중 선택한 키입니다

3. Alternate Key (대체키)
후보키 중 기본키를 제외한 나머지 키입니다 = 보조키

4. Super Key (슈퍼키)
유일성은 만족하지만, 최소성은 만족하지 못하는 키입니다
```

```
Q. DB 정규화란?

A. 자료의 손실이나 불필요한 정보의 도입없이 데이터의 일관성 및 데이터 중복을
최소화하기 위한 기법
최대의 데이터 안정석 확보를 위해 하나의 테이블을 둘 이상으로 분리하는 작업
```

```
Q. DB 정규화의 목적?

A.
1. 중복 데이터를 피해 저장 공간 최소화
2. 데이터 무결성 유지
3. 자료 구조의 안정성 최대화
```

```
Q. 트랙잭션이란?

A. 데이터베이스의 상태를 변화시키기 위해 수행하는 작업 단위
```
```
Q. 트랙잭션 4가지 속성?

A. 
원자성(Atomicity)
트랜잭션이 DB에 모두 반영되거나, 혹은 전혀 반영되지 않아야 됩니다.

일관성(Consistency)
트랜잭션의 작업 처리 결과는 항상 일관성 있어야 한다.

고립성(Isolation)
둘 이상의 트랜잭션이 동시에 실행되고 있을 때, 어떤 트랜잭션도 다른 트랜잭션 연산에
끼어들 수 없습니다.

지속성(Durability)
트랜잭션이 성공적으로 완료되었으면, 결과는 영구적으로 반영되어야 합니다
```

```
Q. DB에서 Index를 사용하는 이유는?

A. 인덱스는 데이터를 논리적으로 정렬하여 검색과 정력 속도를 높이기 위해 사용됩니다
데이터를 특정한 순서로 자주 정렬한다면 인덱스를 사용하기에 적합하지만
삽입 및 변경이 자주 일어나는 데이터에서는 성능이 떨어집니다
```

```

A. 인덱스는 데이터를 논리적으로 정렬하여 검색과 정력 속도를 높이기 위해 사용됩니다
데이터를 특정한 순서로 자주 정렬한다면 인덱스를 사용하기에 적합하지만
삽입 및 변경이 자주 일어나는 데이터에서는 성능이 떨어집니다
```

```
Q. Commit과 Rollback이란??

A.
Commit : 하나의 트랙잭션이 성공적으로 끝났을때, 이를 알려 주기 위해 사용되는 것
Rollback : 하나의 트랜잭션 처리가 비정상적으로 종료되었을 때 원상복귀 하기위해
사용되는 것
```
