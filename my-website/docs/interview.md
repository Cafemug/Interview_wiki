## 기술 면접 예상 질문 대답 요약 정리


```
Q. HTTP 301과 302의 차이는?

A.
- 둘 다 http 리다이렉트이다.
- 차이점은 301은 웹사이드 도메인이 영구이동 했을 때 사용하는 것이고, 302는 요청한 리소스가 임시적으로 새로운 url로 이동했을 때 사용하다. 예를 들어 302는 쇼핑몰에서 임시 품절일 때 사용한다 

```

```
Q. RDMS vs Nosql?

A.
가장 큰 차이는 비관계형 데이터베이스와 관계형 데이터 베이스이다

관계형 데이터베이스
- 관계형 데이터 베이스는 정해진 데이터 스키마에 따라 테이블에 저장된다.
- 그리고 관계를 통해 연결된 여러 개의 테이블에 데이터가 분산되어 저장된다
- 스케일업이 쉽다

비관계형 데이터베이스
- 비관계형 데이터베이스는 스키마와 관계가 없다.
- 스케일 아웃이 쉽다

```

```
Q. ORM이란?

A. 객체와 관계형 데이터베이스의 데이터를 자동으로 매핑(연결)해주는 것을 말한다
```

```
Q. cors란?

A. CORS란 Cross Origin Resource Sharing의 줄임말로, Cross-Site Http Request를 가능하게 하는 표준 규약입니다.
다른 도메인으로부터 리소스가 필요할 경우 cross-site http request가 필요합니다

```

```
Q. crsf?

A. 웹 어플리케이션 취약점 중 하나로 인터넷 사용자(희생자)가 자신의 의지와는 무관하게 공격자가 의도한 행위(수정, 삭제, 등록 등)를 특정 웹사이트에 요청하게 만드는 공격입니다
```

```
Q. 트랜잭션은 무엇인가?

A.
정의
- 트랜잭션은 데이터베이스 시스템에서 병행 제어 및 회복 작업 시 처리되는 작업의 논리적 단위이다.
- 사용자가 시스템에 대한 서비스 요구 시 시스템이 응답하기 위한 상태 변환 과정의 작업단위이다.
- 하나의 트랜잭션은 Commit되거나 Rollback된다.

```

```
Q. https 동작원리에 관해서 아세요?

A.
HTTPS는 HTTP의 하부에 SSL과 같은 보안계층을 제공함으로써 동작한다.

동작원리
•	프로토콜 버전번호 교환
•	양쪽이 알고 있는 pre master secret 키 생성 및 교환
•	양쪽의 신원 인증
•	채널을 암호화 하기 위한 임시 세션 키 생성
```

```
Q. DB 인덱스가 무엇인가?<꼬리질문 O>

A.
- 인덱스는 특정 칼럼값을 가지고 있는 열 혹은 값을 빠르게 찾기 위해서 사용된다
- B+트리로 구현되어있다.
<꼬리질문>그럼 인덱스를 모든 열에 걸면 좋은가?
- 모든 열에 인덱스를 걸게되면 생성속도는 느려진다.
- 인덱스는 모든 열에 거는 게 좋은 게 아니라 조건에 들어가는 부분을 걸면 효과가 좋다
```
